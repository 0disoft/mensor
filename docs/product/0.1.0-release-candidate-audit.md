## Publication Blockers

1. GitHub private vulnerability reporting is enabled, but maintainer
   notification delivery has not been exercised with a test report.
2. The three package identities do not exist in npm yet. Their first `0.1.0`
   publication must be a manual 2FA bootstrap because npm staged publishing
   cannot create a new package.
3. The GitHub `npm-release` environment exists. Each package's stage-only
   trusted publisher binding cannot be configured until that package identity
   exists in npm.
4. The README records the exact future installation command but correctly marks
   it unavailable until registry publication is observed.
5. No npm publication, staged approval, fresh registry-consumer smoke, Git tag,
   or GitHub release exists yet.

These are external release actions, not reasons to add more compiler behavior.
The reviewed local tarball manifest now exists. Do not publish, approve a staged
release, tag, or create a GitHub release until the current candidate passes the
remote gates.
