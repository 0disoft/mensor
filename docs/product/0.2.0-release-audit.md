## Resolved Stage Failure

The first staging attempt, run `29923020216`, stopped before npm staging because
pnpm forwarded a leading `--` argument separator and the release-readiness
parser treated it as an unknown option. Commit
`9a08dc53d40f72dae1165dd0d328609a65b4da25` normalizes one leading separator and
adds focused regression coverage. The successful stage run then repeated the
release check, full aggregate gate, packing, and all three npm stage-publish
jobs rather than resuming after the failed step.

## Compatibility Boundary

- DiagnosticReport v1 remains the default CLI and library output.
- Check Output v2 requires explicit `--report-version 2` or
  `reportVersion: 2` selection.
- Existing exit statuses and v1 parser names remain unchanged.
- Removing the v2 selection is the supported rollback; consumers do not need
  to downgrade the packages.

The normative migration instructions are in `docs/releasing/0.2.0.md`.
