# Migrating to 0.1.0

Version `0.1.0` is the first public Mensor preview. There is no earlier public
package release to upgrade from, but source-checkout consumers may need to
replace provisional workspace package names.

## Package Names

| Provisional source name | Public package |
| --- | --- |
| `@mensor/contract` | `@0disoft/mensor-contract` |
| `@mensor/compiler` | `@0disoft/mensor-compiler` |
| `@mensor/cli` | `@0disoft/mensor-cli` |

`@mensor/fixture-kit` and `@mensor/agent-runner` remain private repository
packages and are not supported consumer APIs.

## CLI Installation

After the packages are present in the npm registry, install the CLI with Node
22 or newer:

```text
pnpm add --save-dev @0disoft/mensor-cli@0.1.0
pnpm exec mensor check . --json
```

The CLI installs `@0disoft/mensor-compiler` and
`@0disoft/mensor-contract` transitively. Consumers that author RouteIndex
artifacts or parse public contracts directly may install the contract package
as an explicit dependency.

## Contract Notes

- Project, feature, diagnostic-report, and RouteIndex schemas remain revision
  1 in this release.
- The optional project `routeIndex` path must point to a canonical, source-bound
  RouteIndex v1 artifact. The compiler validates its source digests and never
  executes an index producer.
- Static `.html` is the only built-in form provider. Dynamic template adapters,
  runtime manifests, autofix, and a production web runtime are not included.
- Public diagnostic compatibility starts at `0.1.0`. Private pre-release
  snapshots and provisional package names are not compatibility anchors.
